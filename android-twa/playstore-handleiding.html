TMS → Google Play Store

Volledige stap-voor-stap handleiding — productie op je eigen Linux cloud

Domein: tms.moveo-bv.nl  ·  Package: nl.moveobv.tms  ·  Methode: TWA (Bubblewrap)

Wat is een TWA? De Android-app is een dun "schilletje" (Trusted Web Activity) rond je bestaande PWA op https://tms.moveo-bv.nl. De app laadt die site fullscreen zonder browserbalk. Inhoudelijke wijzigingen aan de web-app verschijnen automatisch in de app — je bouwt alleen een nieuwe app-bundle bij wijziging van icoon, naam of versienummer.
GOOGLE = doe je in Google-account/Play Console LINUX = doe je op je productieserver (SSH) DNS = doe je bij je DNS-provider LOKAAL/PC = doe je op je eigen werk-pc

2 DNS & domein controleren DNS

De app koppelt aan exact het domein tms.moveo-bv.nl. Dat subdomein moet naar je Linux-server wijzen.

Bij je DNS-provider (bijv. TransIP, Cloudflare, Versio, GoDaddy):

  1. Log in op het DNS-beheer van het domein moveo-bv.nl.
  2. Controleer of er een record bestaat voor het subdomein tms:
    TypeNaamWaardeTTL
    Atms<publiek IP van je Linux-server>3600
    Heb je IPv6, voeg dan ook een AAAA-record toe met je IPv6-adres.
  3. Gebruik je Cloudflare? Zet de proxy (oranje wolk) desgewenst aan — let er dan op dat SSL-modus op Full (strict) staat.
  4. Wacht tot de DNS is gepropageerd (meestal minuten, soms tot een uur).

Controleer vanaf je pc dat het subdomein naar het juiste IP wijst:

nslookup tms.moveo-bv.nl
Het IP in het antwoord moet het publieke IP van je Linux-productieserver zijn.

3 PWA + HTTPS live op Linux productie LINUX

De app kan alleen de browserbalk verbergen als de PWA via geldig HTTPS draait. Jullie productie gebruikt Docker Compose met Nginx Proxy Manager (NPM) voor SSL.

3a. Inloggen op de server

ssh gebruiker@tms.moveo-bv.nl
cd /pad/naar/tmsapp   # map waar docker-compose.yml staat

3b. Laatste code ophalen & stack starten

git pull
docker compose pull
docker compose up -d --build

3c. SSL-certificaat instellen in Nginx Proxy Manager

  1. Open NPM admin: http://<SERVER-IP>:81 (of https://npm.moveo-bv.nl).
  2. Ga naar Hosts → Proxy Hosts en open (of maak) de host voor tms.moveo-bv.nl:
    • Domain Names: tms.moveo-bv.nl
    • Forward Hostname: tms-frontend (containernaam)
    • Forward Port: 80
    • Zet Block Common Exploits aan.
  3. Tabblad SSL: kies Request a new SSL Certificate (Let's Encrypt), vink Force SSL en HTTP/2 aan, accepteer de voorwaarden en sla op.

3d. Controleer dat de PWA en het manifest live zijn

curl -I https://tms.moveo-bv.nl
curl https://tms.moveo-bv.nl/manifest.webmanifest
Je moet een HTTP/2 200 krijgen en het manifest moet geldige JSON teruggeven met "display": "standalone". Pas als dit klopt, ga je verder.

4 assetlinks.json deployen LINUX

In de repo staat al frontend/public/.well-known/assetlinks.json en is frontend/nginx.conf aangepast om dit te serveren. Bij de docker compose up --build uit stap 3 is dit al meegedeployed.

De fingerprint staat nu nog op een placeholder — die vullen we in stap 8 in. Controleer eerst dat het bestand bereikbaar is:

curl https://tms.moveo-bv.nl/.well-known/assetlinks.json
Je krijgt nu JSON terug met REPLACE_WITH_SHA256_FINGERPRINT.... Dat is verwacht; we vervangen die waarde nadat de app gebouwd is (stap 8).

5 Google Play Developer-account GOOGLE

  1. Ga naar play.google.com/console/signup.
  2. Log in met het Google-account dat eigenaar wordt van de app (gebruik een bedrijfsaccount, geen privé-Gmail).
  3. Kies accounttype Organisatie (aanbevolen) en vul de bedrijfsgegevens in. Google vraagt mogelijk om een D-U-N-S-nummer ter verificatie.
  4. Betaal de eenmalige $25 registratiekosten.
  5. Doorloop de identiteitsverificatie (kan 1–3 dagen duren). Pas daarna kun je publiceren.
Plan dit vroeg in: de account- en bedrijfsverificatie bij Google kan enkele dagen duren en blokkeert anders je publicatie aan het eind.

6 Build-tools installeren (op je pc) LOKAAL/PC

De app-bundle bouw je op je eigen werk-pc (Windows), niet op de server.

6a. Java JDK 17

Download & installeer Eclipse Temurin JDK 17. Controleer:

java -version

6b. Bubblewrap CLI

npm install -g @bubblewrap/cli
Bij de eerste build biedt Bubblewrap aan om automatisch de Android SDK en JDK te downloaden — accepteer dat als je geen Android Studio hebt.

7 App bouwen met Bubblewrap LOKAAL/PC

In de repo staat de map android-twa/ met een vooringevulde twa-manifest.json (package nl.moveobv.tms).

7a. Project initialiseren (eenmalig)

cd android-twa
bubblewrap init --manifest https://tms.moveo-bv.nl/manifest.webmanifest
  • Bevestig de waarden (package-naam, app-naam, kleuren komen uit het manifest).
  • Bubblewrap maakt een signing key aan: android.keystore. Je kiest hierbij wachtwoorden.
BEWAAR DE KEYSTORE + WACHTWOORDEN VEILIG (password manager / backup buiten de repo). Raak je deze kwijt, dan kun je nooit meer een update onder dezelfde app publiceren. De keystore staat in .gitignore en mag nóóit in git.

7b. Bouwen

bubblewrap build

Resultaat in de map android-twa/:

BestandGebruik
app-release-bundle.aabUpload je naar de Play Store.
app-release-signed.apkVoor lokaal testen op een toestel.

Bubblewrap toont aan het eind de SHA-256 fingerprint. Noteer deze. Opnieuw opvragen kan met:

keytool -list -v -keystore android.keystore -alias android

8 Fingerprint koppelen & verifiëren LINUX

Nu koppelen we de app aan het domein, zodat de browserbalk verdwijnt.

  1. Open op je pc frontend/public/.well-known/assetlinks.json.
  2. Vervang REPLACE_WITH_SHA256_FINGERPRINT_FROM_BUBBLEWRAP_BUILD door de SHA-256 fingerprint uit stap 7b (formaat AA:BB:CC:...).
  3. Commit & push: 
    git add frontend/public/.well-known/assetlinks.json
git commit -m "chore: add Android TWA fingerprint to assetlinks"
git push
  4. Op de Linux-server opnieuw deployen: 
    ssh gebruiker@tms.moveo-bv.nl
cd /pad/naar/tmsapp
git pull
docker compose up -d --build frontend
  5. Verifieer dat de echte fingerprint nu live staat: 
    curl https://tms.moveo-bv.nl/.well-known/assetlinks.json
  6. Controleer de koppeling met Google's tool: Digital Asset Links tester — vul domein tms.moveo-bv.nl en package nl.moveobv.tms in.
Klopt de fingerprint niet exact, dan toont de app alsnog de browser-adresbalk. Dit is veruit de meest voorkomende fout — controleer dus zorgvuldig.

9 App testen op een toestel LOKAAL/PC

  1. Zet op je Android-toestel Ontwikkelaarsopties en USB-debugging aan.
  2. Sluit het toestel via USB aan en draai: 
    cd android-twa
bubblewrap install
    (of kopieer app-release-signed.apk naar je toestel en installeer handmatig).
  3. Open de app. Controleer:
    • De app opent zonder browser-adresbalk (anders: fingerprint fout, zie stap 8).
    • Inloggen werkt.
    • Push-notificaties vragen netjes om toestemming (Android 13+).

10 Play Console: app aanmaken & vermelding GOOGLE

  1. Open Play Console → Create app.
  2. Vul in: app-naam (TMS), taal (Nederlands), type App, gratis/betaald.
  3. Ga naar Test and release → Production (of eerst Internal testing) en upload app-release-bundle.aab.
  4. Vul de Store listing (winkelvermelding) in:
    AssetVereiste
    App-icoon512 × 512 px PNG
    Feature graphic1024 × 500 px PNG
    Telefoon-screenshotsMinimaal 2 (max 8)
    Korte beschrijvingMax 80 tekens
    Volledige beschrijvingMax 4000 tekens
    Privacybeleid-URLVerplicht (bijv. https://tms.moveo-bv.nl/privacy)
  5. Vul de verplichte vragenlijsten in (linkermenu, Policy & programs / App content):
    • App access: de TMS zit achter login → geef Google een testaccount (gebruikersnaam + wachtwoord), anders wordt de review afgekeurd.
    • Data safety: welke data je verzamelt (login, evt. locatie).
    • Content rating: vragenlijst invullen.
    • Target audience: doelgroep (zakelijk, 18+).
    • Ads: geef aan dat de app geen advertenties bevat.
App access (testlogin) is cruciaal. Omdat je TMS volledig achter een login zit, kan Google's reviewer niets zien zonder account. Zonder geldige testlogin volgt vrijwel zeker een afwijzing.

11 Inleveren voor review & publiceren GOOGLE

  1. Heb je een nieuw persoonlijk account? Doorloop eerst de verplichte gesloten test (12 testers, 14 dagen). Organisatie-accounts mogen direct naar productie.
  2. Controleer dat alle secties een groen vinkje hebben (geen openstaande taken).
  3. Ga naar Production → Releases, klik Review release en daarna Start rollout to Production.
  4. De review duurt doorgaans enkele dagen (eerste keer langer). Je krijgt mail bij goedkeuring of afwijzing.
  5. Na goedkeuring staat de app live op Google Play onder https://play.google.com/store/apps/details?id=nl.moveobv.tms.
Tip: gebruik eerst de Internal testing-track om de app snel (binnen minuten) op een paar toestellen te testen voordat je naar Production gaat.

12 Updates publiceren

Content-updates (nieuwe features in de web-app) vereisen geen nieuwe app-build: na een normale productie-deploy op Linux verschijnt het direct in de app.

Een nieuwe app-bundle is alleen nodig bij wijziging van icoon, naam of native-config:

cd android-twa
# verhoog appVersionCode (+1) en appVersionName in twa-manifest.json
bubblewrap update
bubblewrap build

Upload de nieuwe .aab met dezelfde keystore in Play Console → Production → nieuwe release.
