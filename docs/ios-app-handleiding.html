1. Wat ga je opzetten?

Je TMS-website wordt verpakt in een echte iOS-app. Medewerkers krijgen een uitnodigingslink, installeren via de TestFlight-app van Apple, en hebben daarna een TMS-icoon op hun iPhone — net als elke andere app.

De app is een "wrapper" rond je productie-URL (bv. https://tms.jouwdomein.nl). Updates die jij doet via docker-update.sh verschijnen automatisch bij iedereen — de app zelf hoef je maandelijks alleen opnieuw te laten bouwen om de 90-dagen-vervaltijd van TestFlight te verversen. Dat doet Codemagic automatisch.

Visueel: GitHub commit → Codemagic bouwt iOS-app in de cloud → uploadt naar TestFlight → testers krijgen automatisch update.

2. Wat je nodig hebt

  • Een iPhone of iPad om mee te testen
  • Een Apple ID (gratis, heb je waarschijnlijk al)
  • Een betaalkaart voor het Apple Developer Program ($99/jaar)
  • Een GitHub account (heb je al — deze repo)
  • Toegang tot je domeinprovider (om een privacy policy URL te hosten)
  • ~2 uur tijd voor de eenmalige setup

4. Stap-voor-stap installatie

1

Apple Developer-account aanmaken

  1. Ga naar developer.apple.com/programs.
  2. Klik op Enroll rechts bovenin.
  3. Log in met je Apple ID (of maak er een aan, gratis).
  4. Bij "Entity Type" kies Individual als je het op eigen naam doet, of Organization voor het bedrijf (dan heb je een D-U-N-S nummer nodig — gratis aan te vragen, duurt 5 werkdagen).
  5. Vul je gegevens in, betaal $99 met creditcard of iDEAL.
  6. Apple verifieert je account in 1–3 werkdagen. Je krijgt een e-mail wanneer het klaar is.
Tip: Begin met deze stap meteen — terwijl Apple verifieert kun jij de andere stappen doen.
2

Privacy policy plaatsen

Apple eist een publieke URL met een privacy policy. Simpel HTML-bestandje op je eigen server volstaat.

  1. Maak een tekst met daarin: welke gegevens je verzamelt (e-mail, naam, locatie indien GPS), waarom, hoe lang je ze bewaart, en hoe iemand zijn gegevens kan laten verwijderen.
  2. Plaats hem op bijvoorbeeld https://tms.jouwdomein.nl/privacy.html.
  3. Bewaar de URL — die heb je later nodig.
Hulp nodig? Ik kan een standaard NL/EN privacy policy template voor je genereren in deze repo. Vraag erom.
3

Capacitor toevoegen aan de frontend

Dit is wat de React-app verpakt tot een iOS-app. Ik doe deze stap voor je in de repo; jij hoeft alleen akkoord te geven. Voor de volledigheid: wat er gebeurt:

cd frontend
npm install @capacitor/core @capacitor/cli @capacitor/ios
npx cap init "TMS" "nl.eutransport.tms" --web-dir=dist
npm run build
npx cap add ios

Dit maakt een ios/ map die ik in Git commit. nl.eutransport.tms is de "bundle ID" — een uniek adres voor jouw app. Aanpasbaar als je een andere naam wilt.

4

App registreren in App Store Connect

Pas mogelijk zodra je Apple Developer-account verified is (stap 1).

  1. Ga naar appstoreconnect.apple.com.
  2. Klik My Apps → het blauwe +-icoon → New App.
  3. Vul in:
    • Platform: iOS
    • Name: TMS (of jouw gewenste naam)
    • Primary Language: Dutch (Netherlands)
    • Bundle ID: kies nl.eutransport.tms uit de dropdown (verschijnt automatisch na stap 3)
    • SKU: verzin iets uniek, bv. TMS-001
  4. Klik Create.
5

API-key voor Codemagic genereren

Hiermee kan Codemagic automatisch uploaden naar TestFlight zonder dat jij telkens hoeft in te loggen.

  1. App Store Connect → Users and Access → tab Integrations → Team Keys.
  2. Klik op + om een nieuwe key te maken.
  3. Naam: Codemagic CI. Access: App Manager.
  4. Klik Generate.
  5. Download het .p8-bestand direct — je kunt het maar één keer downloaden!
  6. Noteer de Key ID en de Issuer ID die op deze pagina staan.
Belangrijk: Het .p8-bestand is een geheim. Niet committen naar Git. Bewaar in een wachtwoordmanager.
6

Codemagic account aanmaken en repo koppelen

  1. Ga naar codemagic.io.
  2. Klik Sign up with GitHub en autoriseer toegang tot je repo tmsapp.
  3. In het dashboard: Add application → kies tmsapp → projecttype Capacitor.
  4. Ga naar Teams → Integrations → Apple Developer Portal.
  5. Plak de .p8-key, Key ID en Issuer ID uit stap 5.
  6. Codemagic test de connectie en haalt automatisch je app op.
7

codemagic.yaml in de repo plaatsen

Ik doe deze stap voor je in de repo. Het bestand vertelt Codemagic hoe te bouwen, en regelt:

  • Build bij elke push naar main.
  • Daarnaast 1× per maand een geplande build (zodat de 90-dagen-klok nooit afloopt).
  • Automatische upload naar TestFlight na succesvolle build.
  • E-mail notificatie als de build mislukt.

Jij hoeft alleen de bestaande Apple Developer integratie-naam uit Codemagic in te vullen — dat doe ik samen met je zodra je daar bent.

8

Eerste build starten

  1. In Codemagic dashboard → jouw app → klik Start new build.
  2. Selecteer workflow ios-testflight en branch main.
  3. Wachten: ~10–15 minuten voor de eerste build (latere builds zijn sneller dankzij caching).
  4. Bij succes: de .ipa wordt geüpload naar App Store Connect.
  5. Apple "processed" de build in ~15–30 minuten — je krijgt e-mail.
Eerste build kan falen aan een ontbrekend icoon of een missende permission. Geen probleem — log meldingen door, ik help met fix.
9

TestFlight inrichten

  1. App Store Connect → jouw app → tab TestFlight.
  2. Bij de net-geüploade build: vul kort de Test Information in (welke functies kun je testen, e-mailadres voor feedback).
  3. Maak een Internal Testing Group aan — leden moeten een Apple ID hebben dat onder jouw Apple Developer team valt (jezelf in eerste instantie).
  4. OF: maak een External Testing Group — daar kun je tot 10.000 mensen aan toevoegen via e-mail of een publieke link. Externe testers krijgen wel een eenmalige lichte Apple review (1–2 dagen, vrijwel altijd goedgekeurd).
  5. Verstuur uitnodigingen.
Voor jouw transportbedrijf: kies External Testing met een publieke link. Stuur die link naar je chauffeurs — ze hoeven niet eens op jouw Apple-team te staan.
10

App installeren op de iPhone (testers)

Wat je medewerkers moeten doen:

  1. Installeer de TestFlight-app uit de App Store (gratis, van Apple).
  2. Open de uitnodigingslink/e-mail op de iPhone.
  3. Tik Accept → Install.
  4. TMS verschijnt op het home-screen, herkenbaar aan een klein oranje stipje (geeft aan dat het via TestFlight is — verdwijnt zodra je in de echte App Store gaat).
  5. Inloggen met TMS-account zoals gewoonlijk.
Klaar! Vanaf nu: jij pusht code → Codemagic bouwt → testers krijgen automatisch een nieuwe versie op hun iPhone.

5. Daarna: dagelijks gebruik

Updates uitrollen

Je werkt zoals nu: code aanpassen, git push. Codemagic ziet de push, bouwt automatisch, upload naar TestFlight. Testers krijgen een melding op hun iPhone en updaten met één tik.

Inhoudelijke wijzigingen (frontend/backend)

Omdat de app een wrapper is rond je live TMS-URL, hoeven kleine wijzigingen in de website helemaal geen nieuwe app-build te krijgen — je docker-update.sh op de server is genoeg. Een nieuwe app-build is alleen nodig bij:

  • Native code-wijzigingen (zelden).
  • Iconen of splash-screens veranderen.
  • Permissions toevoegen (camera, locatie, push).
  • Elke 90 dagen — automatisch door de cron.

Build-limiet in de gaten houden

Codemagic gratis: 500 minuten per maand. Eén build ≈ 10–15 minuten. Bij ~3 builds/maand zit je op ~45 minuten — ruim binnen de marge.

6. Veelvoorkomende problemen

Build mislukt: "No matching provisioning profile"

Codemagic moet automatisch een profile maken. Check: in Codemagic → Settings → Code signing identities → "Use automatic code signing" staat aan.

App crash bij opstarten op iPhone

Vaak een Info.plist mis met een verplichte permission-uitleg. Codemagic build-logs tonen welke. Voeg toe in ios/App/App/Info.plist.

TestFlight: "Could not process build"

App-icoon ontbreekt of heeft transparantie. App Store Connect vereist 1024×1024 PNG zonder alpha-channel.

"Geen toegang tot camera/locatie" in de app

iOS vereist toestemming voor elke gevoelige feature. We voegen Capacitor-plugins toe (@capacitor/camera, @capacitor/geolocation) en zetten uitleg in Info.plist.

Build na 90 dagen verlopen

Mag niet gebeuren door de maandelijkse cron — maar als het toch gebeurt: simpel een nieuwe build starten in Codemagic en testers krijgen automatisch update.

Klaar om te beginnen?

De volgorde voor jou:

  1. Start vandaag stap 1 (Apple Developer enrollment) — verifictie duurt 1–3 dagen.
  2. Doe ondertussen stap 2 (privacy policy).
  3. Zeg het seintje, dan voer ik stap 3 en 7 uit in de repo (Capacitor + codemagic.yaml).
  4. Zodra Apple je verified heeft: stap 4 t/m 9 doen we samen.
  5. Stap 10 stuur je naar je collega's.
