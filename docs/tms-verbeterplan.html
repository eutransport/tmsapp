🔴 Kritiek — vóór volgende cloud-release oplossen

Hardcoded DB-wachtwoorden als default in dev/local settings

Verwijder default='tms_password_dev' en default='postgres'. Verplicht .env voor iedere omgeving en fail-fast bij ontbreken.

Backendsettings/development.pysettings/local.py

Klaartekst testwachtwoorden in seed_users.py

Verplaats naar dev-only fixture of genereer random wachtwoord per seed-run. Voorkomt dat een productie-seed een admin met bekend wachtwoord aanmaakt.

Backendseed_users.py

Geen bestandstype-validatie op FileField uploads

Voeg FileExtensionValidator + magic-byte check + Content-Disposition: attachment toe aan invoices, documents, tasks, maintenance, signatures. Nu is een .html/.exe-upload technisch mogelijk.

BackendSecurity

@tanstack/react-query ontbreekt in package.json

Wordt in main.tsx gebruikt maar niet expliciet gedeclareerd. Fresh install kan breken. Toevoegen aan dependencies.

FrontendBuild

Backup wordt gemaakt maar nooit gevalideerd

In update-production-docker.sh na de pg_dump een pg_restore --list of testrestore doen; anders wordt corruptie pas ontdekt bij een echte crash.

DevOpsBackup

Nul frontend tests, minimale backend tests

Vitest + React Testing Library minimaal, Playwright voor login/verlof/factuur flow. Backend: pytest coverage op permissions en multi-tenant scoping.

Testing

Nginx Proxy Manager admin UI open op 0.0.0.0:81

Bind aan 127.0.0.1:81 of intern netwerk. Verplicht wachtwoord wisselen na eerste boot; nu ships met admin@example.com / changeme.

DevOpsSecuritydocker-compose.yml

Ed25519 license private key zonder passphrase

license_manager.py laadt de key via NoEncryption(). Diefstal van het bestand = licenties vervalsen. Passphrase + off-box opslag (KMS, hardware token, of tenminste ge-encrypteerd op VPS).

LicensingSecurity

🟠 Hoog — binnen 1–2 sprints

Geen resource limits op containers

Één runaway Celery-task kan de hele VPS OOM-killen. Voeg deploy.resources.limits toe aan backend, celery-worker, celery-beat, frontend.

DevOps

Sentry SDK geïnstalleerd maar niet geïnitialiseerd

sentry-sdk zit in production.txt; init in production.py ontbreekt. Snelle win voor error-tracking. Overweeg self-hosted GlitchTip voor GDPR.

BackendObservability

django-csp niet actief

Pakket zit in requirements, middleware is niet toegevoegd. Backend stuurt nu geen CSP-header — alleen nginx doet dat, met 'unsafe-inline'.

BackendSecurity

Geen route-level lazy loading in de frontend

Alle 34+ pagina's zitten in de initial bundle. InvoiceCreatePage.tsx is alleen al ~3800 regels. React.lazy() + <Suspense> per route.

FrontendPerformance

Celery tasks zonder retry policy

Eén transient DB-error = herinneringsmail wordt nooit verstuurd. @shared_task(max_retries=3, default_retry_delay=60) + escalatie bij final failure.

BackendReliability

Celery Beat in-memory scheduler

Bij restart worden geplande taken gemist of dubbel uitgevoerd. Overstappen op django-celery-beat (DB scheduler) met migratie.

Backend

Backend healthcheck start-period te kort (5s)

Migrations kunnen 10–30s duren. Zet start-period=60s. Of splits /healthz/live vs /healthz/ready.

DevOpsbackend/Dockerfile

NPM depends_on: service_healthy → deadlock-risico

Als backend traag booted (migrations), blijft NPM in restart-loop en is de maintenance page niet bereikbaar. Zet op service_started.

DevOps

Redis fallback naar localhost in productie

Als REDIS_URL ontbreekt, valt Django terug op redis://localhost:6379/0. Productie: verplicht maken.

Backend

MFA secret in klaartekst in DB

Bestaande EncryptedCharField hergebruiken om de TOTP secret at-rest te versleutelen.

BackendSecurity

Multi-tenant scoping niet consistent

Geen zichtbare "filter altijd op bedrijf/company" op alle ViewSets. Nu waarschijnlijk single-tenant; wordt CRITICAL zodra je meerdere klanten in één DB draait.

BackendArchitecture

:latest tags op NPM en Portainer

Onvoorspelbare updates bij rebuild. Pin naar specifieke versies (bv. jc21/nginx-proxy-manager:2.11.3).

DevOpsdocker-compose.yml

Rollback in update-script niet geïmplementeerd

De rollback() functie in update-production-docker.sh is een stub. Volledig implementeren: containers down, DB restore, media restore, compose restore, up.

DevOps

Per-account login lockout ontbreekt

Nu alleen per-IP throttle (10/min). Botnet met veel IPs kan langzaam password spraying doen. Voeg per-user cooldown na X mislukkingen toe.

BackendSecurity

🟡 Middel — na de hoge prio's

Serializer velden als mfa_enabled, mfa_required niet in read_only_fields

Gebruiker kan die theoretisch patchen. Expliciet read-only zetten.

Backend

SMTP injection risico in planning module

User-supplied SMTP host/from/body zonder validatie. Adressen valideren; body escapen; liever django.core.mail dan raw smtplib.

Backend

JWT in localStorage (obfuscated, niet encrypted)

Bij XSS zijn tokens weg. Overweeg HttpOnly cookie voor refresh token + CSRF token voor mutating requests.

FrontendSecurity

Geen React Error Boundaries

Één throw = witte pagina. Voeg boundary per route toe met terugval-UI.

Frontend

Geen list virtualization

Grote tabellen (invoices, time-entries) kunnen laggen. Gebruik react-window of @tanstack/virtual.

FrontendPerformance

Duplicate i18n keys in nl.json / en.json

TS linter meldt duplicates voor recentActivity, week, totalHours, totalKm. Opruimen.

Frontend

Onvolledige/inconsistente EN translations

Steekproef laat missende keys zien. In dev debug:true op i18next zetten om ze zichtbaar te maken.

Frontend

A11y gaps

Skip-to-content link ontbreekt, iconen-only buttons zonder aria-label, geen consistente focus-management in dialogs.

Frontend

Theme (violet/emerald) niet gekoppeld aan Tailwind primary-*

Themestore verandert kleuren maar veel classes blijven blauw. CSS custom properties koppelen aan Tailwind config.

FrontendDesign

Media op lokaal Docker volume — geen off-site

VPS diskfailure = alle uploads kwijt. Overstap naar S3-compatible (MinIO self-host, Backblaze B2, Wasabi) via django-storages.

DevOpsData safety

Geen data-retention / GDPR purge job

Sessies, oude audit-logs, verwijderde entities blijven eeuwig hangen. Celery Beat job met configureerbare retentie per model.

BackendGDPR

Geen APM / distributed tracing

OpenTelemetry naar self-hosted Grafana Tempo, of Sentry Performance. Nu geen inzicht in trage endpoints.

Observability

Race condition in pakmiddelen scheduler

last_run_at-check is niet atomair; twee beat workers kunnen dubbele mails sturen. DB-lock of Celery task lock (redlock).

Backend

X-Frame-Options overschreven op /media/ in nginx.local.conf

Clickjacking-oppervlak voor PDFs. Terugzetten op SAMEORIGIN of specifiek whitelisten.

DevOpsSecurity

Docker socket gemount naar backend + celery

Read-only mount voorkomt schrijven naar het bestand, niet naar de Docker API. Verwijderen als niet gebruikt; anders via docker-socket-proxy.

DevOpsSecurity

License activate: 10/min throttle en geen exponential backoff

~14.400 pogingen/dag mogelijk. Exponential backoff + audit-log per mislukte activatie.

BackendLicensing

Backup file timeentry_backup_20260331.json in git

Hoort niet in het repo. Verplaatsen naar backup-locatie en .gitignore.

Repo hygiene

🔵 Laag — netjes-om-te-doen

Django/DRF pin op minor version

Django==5.0.x ipv >=5.0,<6.0 voorkomt onverwachte minor-version bumps.

apt-get upgrade -y in backend Dockerfile

OS-CVE's in base image mee-patchen.

pip-audit / safety in CI

Dependency-CVE's automatisch signaleren.

Container image scan (trivy)

Bij elke build de image scannen op known-CVEs.

Runtime-config voor VITE_API_URL

Via window.API_URL in config.js, zodat één frontend image voor meerdere omgevingen kan.

OPERATIONS.md & DISASTER_RECOVERY.md

Runbooks voor logs, restart, health, monitoring, backup/restore procedure.

Correlation-ID middleware

Uniek request-ID per HTTP call, doorreiken naar logs en frontend.

drf-spectacular voor OpenAPI schema

Auto-generated docs + typegeneratie naar frontend mogelijk.

Bundle-analyzer in Vite

rollup-plugin-visualizer om bundle bloat zichtbaar te maken.

RunPython data-migrations reviewen

Alle reverse-functies verifiëren zodat rollback geen data verliest.

Gegroepeerd per domein. Prioriteit binnen elke groep is nice-to-have tenzij anders aangegeven.

DDD/C1B-bestanden parsen voor rij- en rusttijden-controle (EU 561/2006). Bouwt op bestaande tachograph-sync.

Hoge waarde

Per chauffeur dag/week/2-weken rusttijd, max diensttijd, night work limit met live waarschuwingen.

NIS2 audit trail

Bestaande audit-logger uitbreiden naar user/action/before/after/ip/user_agent op alle admin-mutaties.

🗓️ Aanpakvolgorde — sprints van ±1 week

Sprint 1 — hygiëne (weinig risico, veel winst)

  • @tanstack/react-query toevoegen aan package.json.
  • Hardcoded defaults uit dev/local settings en seed scripts weghalen.
  • FileExtensionValidator + magic-byte check op alle FileFields.
  • Backend healthcheck start-period=60s, NPM depends_on: service_started.
  • Resource limits in docker-compose.yml.
  • Sentry init + SENTRY_DSN env var.
  • NPM admin binden op 127.0.0.1:81.
  • Backup verify (pg_restore --list) in update-script.

Sprint 2 — beveiliging & audit

  • Celery task retry policies + django-celery-beat.
  • MFA secret at-rest encryption via bestaande EncryptedCharField.
  • django-csp middleware activeren; permissive start, aanscherpen.
  • Per-account login lockout naast IP throttle.
  • Frontend Error Boundaries + route-level lazy loading.
  • Duplicate i18n keys opruimen.

Sprint 3 — data safety & observability

  • Media naar object storage (MinIO self-host of Backblaze) via django-storages.
  • Automated pg_dump naar remote (Restic/rclone) + retention.
  • APM: OpenTelemetry naar self-hosted Grafana Tempo, of Sentry Performance.
  • Retention/purge job (Celery Beat).

Sprint 4 — feature-keuze

Kies 1–2 items uit de feature-tab die passen bij je klanten. Persoonlijke tip:

  • Tachograaf DDD-analyse (A) — compliance-winst en direct nuttig.
  • PEPPOL e-factuur (B) — regelgeving loopt op je af, nu al klaarzetten.
  • Publieke ETA-link (C) — snel te bouwen op bestaande signed-URL infra, hoge klantwaarde.
  • Chauffeurs-modus PWA (G) — dagelijkse gebruikswaarde, verhoogt adoption.

Beperkingen om rekening mee te houden op de VPS/Docker stack

  • CSP zonder 'unsafe-inline' vraagt Vite-hashing van inline scripts en test op dynamische theme-kleuren.
  • HttpOnly refresh-cookie vraagt herschrijven van refresh-flow + CSRF-tokens (1–2 dagen werk).
  • Multi-tenant achteraf invoeren = bulk-migratie (welke row hoort bij welk bedrijf) + audits op alle get_queryset-overrides.
  • Object storage vraagt migratie van bestaande media-URLs.
  • Route-optimalisatie: GraphHopper is self-host op dezelfde VPS haalbaar; Google Routes vraagt betaald account.
  • PEPPOL vereist een access point (Storecove of Digipoort) — technisch simpel, wel abo/licentie.
  • Docker Compose deploy.resources werkt met moderne Docker (v25+) prima buiten Swarm.
